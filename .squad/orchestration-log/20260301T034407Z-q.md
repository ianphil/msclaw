## Key Changes to Bootstrap Plan

### Architecture Insight: `.ainotes/` is the Memory System

The guide defines:
- **memory.md** — Curated long-term memory, read every session
- **rules.md** — Mistake journal, one-liners that compound
- **log.md** — Raw chronological log, append-only
- Consolidation every ~14 days

Impact: T3 (Validator) and T5 (Scaffold) must use `.ainotes/` not `.working-memory/`

### New Understanding: Host Repo vs Mind Directory

- **Mind** = SOUL.md + .ainotes/ + IDEA folders (domains/, initiatives/, expertise/, inbox/, Archive/)
- **Host repo** = `.github/agents/{name}.agent.md` (operating instructions) + `.github/skills/{name}/SKILL.md` (reusable workflows)

Phase 1a creates the mind. Phase 1b adds host repo awareness (future).
